## 매칭 시작창 중심 전면 교체 계획 (매칭 탭 전체교체 + 채팅 UI 오류 포함)

### 요약
- 기존 `MatchScreen`의 3탭 구조를 제거하고, `매칭 시작` 단일 플로우로 교체합니다.
- 카드 타입을 `개인`과 `방(2인 팀)`으로 명확히 분리해 표시 규칙/테두리 규칙을 반영합니다.
- 풀 생성은 `90~100:1명, 80~90:2명, 60~80:2명` 목표로 랜덤 샘플링하되 부족 시 하위 밴드로 보충합니다.
- 채팅은 SSE 기반 3일 보관·수신 즉시 폐기를 유지하고, UI 동기화 오류까지 포함해 수정합니다.
- 방 매칭 확정은 `대표자 확인` 방식으로 처리합니다.

### 핵심 변경 사항
- 프론트 매칭 탭 전면 교체:
  - `MatchScreen`을 “매칭 시작 화면” 단일 플로우로 재구성(풀 목록, 상세 요약, 채팅 진입, 상태 배지 포함).
  - 개인 카드: `상대 닉네임 + 일치율`, 기존 테두리 유지.
  - 방 카드: `닉네임 2개 + 공통 일치율 + 개별 일치율`, 진한 파랑 테두리, `OOO, OOO의 방` 표기.
- 상세 화면 표시 축약:
  - 클릭 시 세부 조건(diff/세부 항목)은 숨기고 대주제 단위(`취침 및 생활 패턴`, `생활 습관`, `친밀도`)를 `상/중/하`만 표시.
- 백엔드 풀 샘플링 로직 보정:
  - 현재 정렬 상위 고정 선발을 밴드별 랜덤 선발로 변경.
  - 90대/80대 부족 시 80대 추가, 그마저 부족 시 70대/60대 순으로 5명까지 보충(불가 시 미충원 허용).
- 3인실 이상 2인 팀 카드 생성 규칙 유지/명확화:
  - 최초 2명 매칭 시 2인 평균으로 재진입 카드 생성.
  - 후보 타입 `room` payload에 `member_names`, `member_scores`, `shared_score`를 표준화.
- 채팅/세션 상태 전파 정합성 수정(UI 오류 포함):
  - 스레드 종료 사유(`already_matched`, `no_response`, `rematch`, `cancelled`) 즉시 반영.
  - SSE 수신 이벤트 수신 시 메시지 리스트/스레드 리스트 동기화 누락 보강.
  - 읽음 처리 시 서버 폐기와 UI 제거 타이밍 일치화.
- 매칭 성사/차단 플로우:
  - 성사 시 다른 채팅방 전부 시스템 메시지 `사용자가 이미 매칭됨` 전송 후 입력 차단.
  - 5개 전부 종료 시 재매칭 가능 상태 노출.
  - 2일 무응답 자동 취소, 성사 후 3일 쿨다운, 재매칭 시 상대에게 알림 이벤트 유지.
- 방 매칭 확정 정책(대표자 확인):
  - 방 카드 세션에서 방 구성원 중 대표 1인의 확인으로 성사 처리.
  - DB/API에 대표자 식별 필드(또는 규칙: 첫 번째 member) 추가 및 확인 로직 분기.
- 인터페이스/API 변경(중요):
  - `GET /api/match/pool` 응답 스키마를 `candidate_type`별 필수 필드로 고정.
  - `POST /api/match/confirm`에 `room_confirm_mode=delegate` 동작 반영(하위 호환 유지).
  - 필요 시 `match_sessions`에 `delegate_uid`(또는 동등 의미) 컬럼 추가.

### 테스트 계획
- 풀 생성:
  - 90/80/60 밴드 충족 시 정확히 1/2/2 분포.
  - 90 없음 → 80 보충, 80도 부족 → 70/60 보충 케이스.
  - 하드블록 제외 후 5명 미만 허용 케이스.
- 카드/UI:
  - 개인/방 카드 텍스트 및 테두리 색상 검증.
  - 상세에서 대주제 `상/중/하`만 표출되는지 검증.
- 채팅/SSE:
  - 미수신 3일 보관 후 만료 삭제.
  - 수신+읽음 즉시 서버 삭제 및 클라이언트 반영.
  - 성사 후 타 스레드 시스템 메시지 삽입/입력 차단.
- 세션 상태:
  - 2일 무응답 자동 취소.
  - 5개 스레드 모두 종료 시 재매칭 가능.
  - 성사 후 3일 쿨다운, 재매칭 알림 이벤트 송신.
  - 방 카드 대표자 확인 시 성사, 비대표 확인 시 대기 유지.

### 가정 및 기본값
- “매칭 시작 창”은 기존 `매칭 탭 전체`를 대체하는 것으로 확정.
- 방 매칭 확정은 `대표자 확인` 1회로 성사.
- “대화 기능 오류 재생”은 핵심 흐름 + 채팅 UI 상태 동기화 오류까지 포함.
- 기존 레거시 탭(`top/pairs`)은 기본 UX에서 제거하고, 호환 API는 당분간 유지.
